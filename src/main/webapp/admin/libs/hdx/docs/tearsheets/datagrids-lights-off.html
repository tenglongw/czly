Datagrid Basics

Datagrids Basics

HDX Datagrids take advantage of the Datatables jQuery plugin.

Datagrids come pre-styled and should fully support the Datatables.net API. This includes loading data with JavaScript Arrays, AJAX and progressive server updates. See datatables.net for more information about datatables configuration.

General notes

  • Add class=”outlier” to elements to mark a cell as an outlier.
  • Any input elements with an attribute data-filter-table=”table-name” (where table-name is the data-table-name of an IIDS data table) will become filters using the standard datatables.net fnFilter() capability. See also the “Simple Column Filtering” example.
  • To disable the floater on rollover set the useFloater configuration option to false. e.g. $('.my-table').iidsBasicDataGrid({ useFloater: false });
  • Responsive design
    • To enable responsive breakpoints set the isResponsive configuration option to true. e.g. $('.my-table').iidsBasicDataGrid({ isResponsive: true });
    • When isResponsive is enabled, the number of columns displayed in the table depends on the available screen space. By default, a smartphone will display ~2-3 columns, while an expanded desktop browser displays the full set. This is accomplished by assigning semantic classes to the column headings that indicate which data values take precedence (essential vs optional). If a column does not contain an essential or optional class it will always have the lowest precedence.
    • While isResponsive is enabled you can prevent certain breakpoints from firing by using the excludes option. For example excludes: { phone: true }} will prevent the component from dropping optional columns on phones. The supported excludes are phone, portraitTablet and landscapeTablet.
  • Plugins
    • In the base “datatables” JQuery plugin, the notion of “datatables plugins” is supported with a plugin API. I few plugins ship with the IIDx.
    • To use a plugin, the plugin JavaScript must be included on the page and the plugin enabled in a datagrid instance. In the base “datatables” JQuery plugin, datatables plugins are enabled by adding the plugin code to the “sDom” configuration parameter. This is messy as “sDom” contains a lot of settings that would need to be repeated. IIDx datagrids provides a separate “plugins” configuration parameter that expects an Array of plugin codes, e.g. $('.my-table').iidsBasicDataGrid({plugins: ['R']});

Column Reordering

Column reordering

A Datagrid using interactive column reordering.. Drag any column using the mouse, a blue line to the right of the nearest column will indicate where the dragged column will be placed when dropped.

Example
Usage

Markup:

<table cellpadding="0" cellspacing="0" border="0" class="table table-bordered" data-table-name="dt-col-reorder-example"></table>

Script:

//Column reordering
$('table[data-table-name="dt-col-reorder-example"]').basicdatagrid({
	'aoColumns': [
		{ 'sTitle': 'Rendering engine', 'mData': 'engine', 'sClass': 'essential'},
		{ 'sTitle': 'Browser', 'mData': 'browser', 'sClass': 'essential'},
		{ 'sTitle': 'Platform(s)', 'mData': 'platform', 'sClass': 'optional'},
		{ 'sTitle': 'Engine version', 'mData': 'version'},
		{ 'sTitle': 'CSS grade', 'mData': 'grade', 'sClass': 'essential'}
	],
	'useFloater': false,
	'sAjaxSource': '../assets/data/ao.browsers.json',
	'plugins': ['R'] //enable the col-reorder plugin (assumes 'col-reorder-amd' is on the page)
});

Notes:

  • We use the DataTables project's "ColReorder" plugin for this functionality.
  • You must include the plugin library on the page. The requirejs id for this in HDX is 'col-reorder-amd'
  • Configure according to the plug-in's documentation with the following changes.
    • Datatable "plugins" have been moved out of the "sDom" settings parameter into a separate "plugins" settings Array parameter, so enable the plugin by setting 'plugins': ["R"] on your table
Script Requirements:
  • datagrids
  • col-reorder-amd

Simple Column Filtering

Column filtering

A Datagrid using interactive column filtering.

Note: There is a bug with the 3rd party datatables component. On the iPad, when attaching a touch event to the body via the body tag or Javascript/jQuery, this text field input is blocked from allowing input
Example
Usage

Markup:

<table cellpadding="0" cellspacing="0" border="0" class="table table-bordered" data-table-name="dt-col-filter-example"></table>

Script:

//Filtering
$('table[data-table-name="dt-col-filter-example"]').basicdatagrid({
  'aoColumns': [
    {'sTitle': 'Rendering engine', 'mData': 'engine', 'sClass': 'essential'},
    {'sTitle': 'Browser', 'mData': 'browser', 'sClass': 'essential', 'filter':true},
    {'sTitle': 'Platform(s)', 'mData': 'platform', 'sClass': 'optional', 'filter': true},
    {'sTitle': 'Engine version', 'mData': 'version', 'filter':true, 'filterPlaceholder':'Filter by this field'},
    {'sTitle': 'CSS grade', 'mData': 'grade', 'sClass': 'essential', 'filter':true}
  ],
  'sAjaxSource': '../assets/data/ao.browsers.json',
  'useFloater': false,
  'isResponsive': true
});

Notes:

  • Use the "filter:true" and, optionally, "filterPlaceholder:..." members on each aoColumns element for the columns for which you want to enable column filtering..
Script Requirements:
  • datagrids

Column Hiding

Column hiding

A Datagrid using interactive column hiding.

Example
Usage

Markup:

<div class="pull-right">
  <div class="col-vis-dropdown-container"></div>
  <div class="input-append">
    <input type="text" class="input-sm search-query" data-filter-table="dt-ajax-array"><button class="btn btn-icon"><i class="icon-search"></i></button>
  </div>
</div>
<table cellpadding="0" cellspacing="0" border="0" class="table table-bordered" data-table-name="dt-col-hiding-example"></table>

Script:

//assume 'datagrids/datagrids-col-vis' is on the page...
$('table[data-table-name="dt-col-hiding-example"]').basicdatagrid({
  'aoColumns': [
    {'sTitle': 'Rendering engine', 'mData': 'engine', 'sClass': 'essential' },
    {'sTitle': 'Browser', 'mData': 'browser', 'sClass': 'essential' },
    {'sTitle': 'Platform(s)', 'mData': 'platform', 'sClass': 'optional' },
    {'sTitle': 'Engine version', 'mData': 'version' },
    {'sTitle': 'CSS grade', 'mData': 'grade', 'sClass': 'essential' }
  ],
  'sAjaxSource': '../assets/data/ao.browsers.json',
  'useFloater': false,
  "plugins": ["C"], //enable the Column Vis plugin (assumes datagrids/datagrids-col-vis.js is on the page)
  "oColVis": { //configure the column vis plugin
    "aiExclude": [ 1 ], // don't allow 2nd column to be hidden
    "colVisDropdownSelector": ".col-vis-dropdown-container" //put the column hiding control inside the element at this selector
  }
});

Notes:

  • We forked the DataTables project's "ColVis" plugin for this functionality.
  • You must include the plugin library "[datagrids root]/js/datagrids/datagrids-col-vis.js" on the page. The requirejs id for this in HDX is 'datagrids/datagrids-col-vis'
  • Configure according to the original plug-in's documentation with the following changes.
    • Datatable "plugins" have been moved out of the "sDom" settings parameter into a separate "plugins" settings Array parameter, so enable the plugin by setting 'plugins': ["C"] on your table
    • Place the column hide dropdown inside any element consistent with design guidelines by specifying a "colVisDropdownMenuSelector", which is a valid JQuery selector.
    • The notion of "column groups" to show/hide has been removed
Script Requirements:
  • datagrids
  • datagrids/datagrids-col-vis

Row Grouping

Row grouping

A Datagrid using interactive row grouping.

Example
Usage

Markup:

<div class="row-grouping-dropdown-container"></div>
<table cellpadding="0" cellspacing="0" border="0" class="table table-bordered" data-table-name="dt-row-grouping-example"></table>

Script:

//assume 'datagrids/datagrids-row-grouping' is on the page...
$('table[data-table-name="dt-row-grouping-example"]').basicdatagrid({
  'aoColumns': [
    {'sTitle': 'Rendering engine', 'mData': 'engine', 'sClass': 'essential'},
    {'sTitle': 'Browser', 'mData': 'browser', 'sClass': 'essential'},
    {'sTitle': 'Platform(s)', 'mData': 'platform', 'sClass': 'optional'},
    {'sTitle': 'Engine version', 'mData': 'version'},
    {'sTitle': 'CSS grade', 'mData': 'grade', 'sClass': 'essential'}
  ],
  'sAjaxSource': '../assets/data/ao.browsers.json',
  'useFloater': false,
  'plugins': ["G"], //enable the row grouping plugin (assumes datagrids/datagrids-row-grouping.js is on the page)
  "bPaginate": false, // pagination is not compatible with row grouping
  "rowGrouping": { //configure the row grpuping plugin
    rowGroupingDropdownSelector: ".row-grouping-dropdown-container" //tell the plugin where to put the row grouping control
  }
});

Notes:

  • You must include the plugin library "[datagrids root]/js/datagrids/datagrids-row-grouping.js" on the page. The requirejs id for this in HDX is 'datagrids/datagrids-row-grouping'
  • Datatable "plugins" have been moved out of the "sDom" settings parameter into a separate "plugins" settings Array parameter, so enable the plugin by setting 'plugins': ["G"] on your table
  • Place the row grouping dropdown inside any element consistent with design guidelines by specifying a "rowGroupingDropdownSelector", which is a valid JQuery selector.
  • Pagination must be disabled when row grouping is enabled
Script Requirements:
  • datagrids
  • datagrids/datagrids-row-grouping

      <th class="essential">Baz</th>
      <th class="optional">Qux</th>

      <td>Hello2</td>

  • Add class="outlier" to <td> elements to mark a cell as an outlier.
  • To disable the floater on rollover set the useFloater configuration option to false. ex: basicdatagrid({ useFloater: false });
  • To enable responsive breakpoints set the isResponsive configuration option to true. ex: basicdatagrid({ isResponsive: true });

This is accomplished by assigning semantic classes to the column headings that indicate which data values take precedence (essential vs optional). If a column does not contain an essential or optional class it will always have the lowest precedence.
  • While isResponsive is enabled you can prevent certain breakpoints from firing by using the excludes option. For example excludes: { phone: true }} will prevent the component from dropping optional columns on phones. The supported excludes are phone, portraitTablet and landscapeTablet.
