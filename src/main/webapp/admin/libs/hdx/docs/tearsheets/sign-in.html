Sign In

Basics

HDX Sign In

Sign In component come pre-styled and pre-structured with header and content of the component. It supports few optional attributes which user can use to edit the component as per their requirements.

Sign In optional attributes are as follows header, app-name, forgot-pwd and error-block.

Optional Sign In attributes

  • header: Set header = true/false to show or hide the header. Sign In header includes the GE logo, Application name and GE Healthcare brand name. Entire header can be removed by setting this attribute to false.
  • app-name: This attribute can be used to add an application name to the header.
  • forgot-pwd: Set forgot-pwd = true/false to show or hide the 'forgot your password' link. On clicking this link, forgot password window will be opened. User can use this to reset the password.
  • error-block:This attribute can be used to add sign-in error message. When server cannot authenticate user due to invalid username or password, it may throw an error or an exception based on authentication error handling code. To display this error on Sign In screen, the error-block attribute can be linked to the error message received from server.

Sign In (with no error message)

Open fullscreen example.

Description

A Sign In screen with username and password information. This example displays header and forgot password link.
Usage
            <div class="container col-sm-4 signin-container">
        		<div class="signin" header="true" app-name="HDX Application" forgot-pwd="true" error-block="">

        		</div>
        	</div>
        	
            <script type="text/javascript">
            require(['signin'], function() {
              $('.signin').signinconstruct();
            });
            </script>
Script Requirements:
  • ge-bootstrap
  • jquery, require
  • signin
