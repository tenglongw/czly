Popovers

Markup:

          <a class="btn btn-danger popover-target" data-trigger="click"
          data-content="And here's some amazing content. It's very engaging.
          right?" data-title="A Title">click for popover</a>

Script:

          $(…).popover(options);

In general, clicking outside a popover will not close the popover. To have the popover dismiss on next mouse click, use focus trigger and include tabindex attribute. 

          <a class="btn btn-danger popover-target" data-trigger="focus" tabindex="0"
          data-content="And here's some amazing content. It's very engaging.
          right?  This popup will dismiss on next mouse click" data-title="A Title">click for dismissible  popover</a>

Options:

Name Type Default Description
animation boolean true Apply a css fade transition to the tooltip
html boolean false Insert HTML into the popover. If false, jquery's text method will be used to insert content into the DOM. Use text if you're worried about XSS attacks.
placement string|function 'right'

how to position the popover - top | bottom | left | right | auto.

When "auto" is specified, it will dynamically reorient the popover. For example, if placement is "auto left", the popover will display to the left when possible, otherwise it will display right.

When a function is used to determine the placement, it is called with the popover DOM node as its first argument and the triggering element DOM node as its second. The this context is set to the popover instance.
selector string false if a selector is provided, tooltip objects will be delegated to the specified targets
trigger string 'click' How popover is triggered - click | hover | focus | manual. You may pass multiple triggers; separate them with a space.
title string | function ''

Default title value if title attribute isn't present

If a function is given, it will be called with its this reference set to the element that the popover is attached to.
content string | function ''

Default content value if data-content` attribute isn't present

If a function is given, it will be called with its this reference set to the element that the popover is attached to.
delay number | object 0

Delay showing and hiding the popover (ms) - does not apply to manual trigger type

If a number is supplied, delay is applied to both hide/show

Object structure is: delay: { show: 500, hide: 100 }
container string | false false

Appends the popover to a specific element container: 'body'

This option is particularly useful in that it allows you to position the popover in the flow of the document near the triggering element - which will prevent the popover from floating away from the triggering element during a window resize.

All of the above options may be applied independently as data- attributes.

NOTE: For performance reasons, popovers are opt-in. They must be manually initialized to function.

NOTE: hds.js includes additional code snippets to allow easy dismissal of popovers and to prevent default click behaviors.

        $(...).popover({ trigger: 'click' }).on('click', function (e) {
            e.preventDefault();
        });
        $(...).popover().on('blur', function (e) {
            $(this).popover('hide');
        });

NOTE: See Bootstrap documentation for a full list of options and events for popovers.

Script Requirements:
  • bootstrap/bootstrap-popover
  • bootstrap/bootstrap-transition

Usage

The content to be displayed in the richpopover may be specified using data-content-selector in the markup:

        <a href="#" id = "rich-popover-1" class="btn btn-danger popover-target" data-html="true" data-popover-type="rich" data-content-selector="#popover1-content-wrapper" data-placement="right">click for popover 1</a>
        <div id="popover1-content-wrapper" style="display: none">
            <h2>Suspendisse</h2>
            <p>
                Nullam quis risus <a href="#">eget urna</a> mollis ornare vel<br />
                eu leo. Cum sociis natoque penatibus et magnis<br />
                dis parturient montes, nascetur ridiculus mus.<br />
                Nullam id dolor id nibh ultricies vehicula.
            </p>
            <br />
            <p>
                <button class="btn btn-rich">Button One</button>
                <button class="btn btn-rich">Button Two</button>
            </p>
        </div>

and initializing the richpopover using:

        $('#popover1').richPopover();

It is also possible to not use the data-content-selector in the markup: 

        <a href="#" id = "rich-popover-2"  class="btn btn-danger popover-target" data-html="true" data-popover-type="rich"  data-placement="right">click for popover 2</a>
        <div id="popover2-content-wrapper" style="display: none">
            <h2>Suspendisse</h2>
            <p>
                Nullam quis risus <a href="#">eget urna</a> mollis ornare vel<br />
                eu leo. Cum sociis natoque penatibus et magnis<br />
                dis parturient montes, nascetur ridiculus mus.<br />
                Nullam id dolor id nibh ultricies vehicula.
            </p>
            <br />
            <p>
                <button class="btn btn-rich">Button One</button>
                <button class="btn btn-rich">Button Two</button>
            </p>
        </div>

and pass richpopover content as an option:

        $('#popover2').richPopover({
          content: function() {
            return $('#popover2-content-wrapper').html();
          }
        }).on('click', function(e) {
          e.preventDefault();
        });
