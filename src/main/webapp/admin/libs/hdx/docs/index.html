Masthead and Navigation Patterns

The Masthead and Navigation are a consistent, global framework throughout HDx. The Masthead provides global controls and alerts, and the primary and secondary Navigation communicate to the user where they are within the system.

The Masthead and Global Navigation are consistent at the top of every screen across HDx. The masthead is the topmost bar and holds elements pertaining to the entire system. This includes system alerts, global search, and system settings. The primary navigation appears under the masthead, and allows for navigation through the different screens within the system. The secondary navigation sits below, and holds elements pertaining to the current screen, including breadcrumbs, screen title, secondary tabs, and contextual controls for the content on the screen.

Sample Application

Dashboard

The Dashboard is an easily scannable, overarching summary of elemental information generated by the application. It acts as a hub of information and communication, and provides both previews and shortcuts to more detailed components in the application. The Dashboard loads at application launch and is personalized for each user. Itâ€™s populated with a set of widgets focused on communication which may include monitoring status and the activity of patients, devices, and staff. The Dashboard can be customized by rearranging and adding new widgets specific to the user's needs.

Patient List

Devices and/or patients are indexed in a filterable, sortable Data Table. Items are listed with identifying information, such as status, date, and ID numbers. Filters appear in an omnipresent Filter Toolbar at the top of the screen, and reveal query-building controls that allow the user to reveal, hide, or highlight data grid contents based on text matching conditions. The filter labels are progressively populated with selection and double as labels for the table content. Multi-select filters reveal nested accordion drop downs to allow for multiple, categorized filtering.

When selected, rows in the data table expand to reveal a preview, called the Inspector View, which provides a snapshot of the data associated with the row's content. The preview presented in the Inspector View pulls information from a separate, more detailed screen within the system, such as a Detail or Analytics screen. If more detailed information is needed, selecting the View More Details button in the Inspector View opens the relevant screen.

Patient Detail

The Detail screen holds detailed information about a patient or device. This includes identifying information, current asset status, recent events, notes, configuration data, and snapshots of patient or device metrics. A patient or device summary banner provides a snapshot of statistics. Some modules in the Detail screen include links to view more details, which opens corresponding supplemental screens and are navigated through progressively populated breadcrumbs.

Analytics

An Analytics screen represents a place where users can analyze and compare data to identify trends and insights. It provides detailed charts and visualizations of patient and device data. These visualizations may be defined by unit measurements, timelines, or device type. The charts and graphs are manipulated by filters, view selectors, scrubbers, and date pickers. Multiple tabs allow for easy data comparison, and and expandable data table allows for an alternate view of the visualized data.

Reports

The Reporting page acts as a hub for accessing past reports and generating new ones. A report archive presents sortable thumbnails of past reports which, when selected, display previews of the reports below. The report preview presents descriptions, multi-user comments, and contextual controls. Reports, in this context, provide a living document that users can share and discuss virtually.

OffCanvas Navigation Menu ( In-progress )

Demonstration implementation of responsive navigation menu which addresses the need to provide a better intuitive solution for navigating a multi-layer menu once it reaches a specific break point that is not optimal for the current navigation state. The design is intended to be a solution for only the portrait orientation on the iPad, when both the landscape and portrait orientations are supported.
